# Contributing to dsxtool

To maintain code integrity and development efficiency, follow these guidelines:

### Development Environment
* **Languages:** :Bash.
* **OS:** Linux. (All distros)
* **Code Standard:** Use consistent indentation and English commit messages.

### Submission Process
1. Fork the repository.
2. Create a feature branch (`git checkout -b feature/new-tool`).
3. Ensure scripts pass validation/lint checks (for example: `bash -n` and `shellcheck`).
4. Open a Pull Request detailing the changes.

### Critical Rules
* No bloatware.
* Performance-oriented and minimalist code.
* Document all new functions.
